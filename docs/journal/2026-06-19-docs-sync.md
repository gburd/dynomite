## Doc-link fixes (Part 1)

`RUSTDOCFLAGS="-D warnings" cargo doc --workspace --no-deps
--features riak` was failing on a batch of broken / private /
redundant intra-doc links. Fixed every one by correcting or
de-linking; no `#[allow(...)]` added. Files and the reason each
link broke:

* gen-fsm/transition.rs: `EventType::Enter` does not exist (state
  entry routes through `FsmHandler::on_enter`).
* dyn-sup/atomics.rs, throttle-core/lib.rs: `loom::sync::atomic` /
  `loom` not in the doc graph -> plain backticks.
* tre-sys/lib.rs: public docs linked private `raw` module -> link
  the public re-exports instead; `raw` itself as plain text.
* dynomite-text/lib.rs: `persist` module is `cfg(feature=noxu)`,
  not built here -> plain text.
* dyn-hashtree/lib.rs: private `HashTree::segments` field -> public
  `segment_count`.
* dynomite-vec: `crate::index::TurboTable` is actually
  `crate::turbo_index::TurboTable`; `f16::from_f32` -> `half::f16`;
  `api` module is `cfg(feature=http)`; `MemoryStore`/`VectorTable`
  renamed to `MemoryBackend` / plain text.
* dynomite: `collect_routable`, `DYNIAK_SUPPORTED`,
  `murmur3_x64_128` are private; `dyniak::*` / `dynomited::*` are
  not deps of the engine; `Any::downcast_ref` is an inherent method
  not on the trait; `crate::stats::histogram` is a private mod
  (-> `crate::stats::Histogram`); 5 redundant-explicit-link-target
  warnings in dispatch/embed/msg fixed by dropping the explicit
  target now that the item is re-exported in scope.
* dyniak: `RpbPing` -> `RpbPingReq`/`Resp`; `ServeConfig` and
  `from_pool` do not exist; `request_tx`/`Bucket::directory`/
  `MAX_RHO` are private; `resolve`/`Scheduler::tick` ->
  `Self::resolve`/`Self::poll`; `WasmModuleStore` is
  `cfg(feature=wasm)`; `super::HandoffHandler` does not exist;
  `KeyFun::route_bytes` (was `Self::` on the error enum);
  `dynomite::*` / `TopologySnapshot` not in scope.
* dyniak-bench: `record_ok`/`record_err` live on `WorkerHandle`,
  `flush` on `StatsAggregator`.
* dyn-admin/dynomited: `dynomite::*` cross-crate links and private
  `ListenerHandles` / `backend_supervisor` -> plain text.

Result: `cargo doc -D warnings --features riak` is clean.
