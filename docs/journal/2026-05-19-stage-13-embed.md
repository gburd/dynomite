## Goal

`docs/book/src/embedding/`: typed builder, runtime handle, event
stream, hook traits, and default impls. The embed API is the

## Design choices

### Object-safe async via `BoxFuture`

`Datastore::dispatch`, `MetricsSink::emit`, and friends use
`BoxFuture<'a, Result<T, E>>` rather than the `async fn`
return-position trait shape Rust 1.75+ allows. Reason: object
safety is a hard requirement (the builder takes `Box<dyn
Datastore>` etc.); without `async_trait` on the workspace
dependency list the boxed-future pattern is the cleanest path.
Type alias `BoxFuture<'a, T>` is exposed publicly so embedders
write the same shape as the in-crate impls.

### In-process peer registry

`inject_request` needs to honour `DispatchPlan::Replicas`
without wiring a real dnode forward path through the
connection-pool / mbuf chain (that work belongs to the live
runtime stages, not the embed facade). A process-wide static
registry keyed on `dyn_listen` `SocketAddr` lets the embed
surface forward to a co-located peer's `Datastore` directly.
Production cross-process traffic is unchanged; the registry is
never consulted unless the target peer's address resolves to a
`SocketAddr` that another in-process server registered.

Recorded as a Deviation in `docs/parity.md`.

### `RustCryptoProvider` over `OpensslCryptoProvider`

The design doc names the default crypto provider
`OpensslCryptoProvider`. The workspace migrated to RustCrypto in
Stage 6 (`docs/journal/2026-...-stage-06-crypto.md`); the embed
default keeps the type name aligned with the actual backend. The
trait surface is identical.

### `LoggingMetricsSink` over `PrometheusMetricsSink`

Same reasoning as Stage 13's brief: shipping a Prometheus
exporter would add a workspace dependency, which is governed by
PLAN.md Section 2 plus a journal entry. `LoggingMetricsSink` is
dependency-free, exercises the trait surface, and documents the
Prometheus shape in the rustdoc + design doc for the embedder
who needs it. Recorded as a Deviation.

### Custom-transport example uses `tokio::io::duplex`

`embedded_with_custom_transport.rs` shows the `Transport` trait
shape with a dependency-free in-memory `DuplexStream`. The
inline doc note points at `tokio_rustls::TlsStream` for
production wiring; the trait surface does not change.

### Builder validates `ConfPool` directly

`ServerBuilder::build` calls `ConfPool::apply_defaults` +
`ConfPool::validate(pool_name)` rather than serialising the pool
to YAML and re-parsing through `Config`. The validation surface
is identical (the same helper backs `Config::validate`); the
direct path avoids an asymmetric serde shape on `ConfDynSeed`
(serialises as `pname` only, deserialises full form).

## SIGHUP / `dynomite_reload_conf` parity

The C reference's `dynomite_reload_conf` (in
`_/dynomite/src/dynomite.c`) re-reads the YAML from disk and
swaps the global pool reference under a lock. Stage 13 mirrors
this with `ServerHandle::reload(Config)`:

1. Run `Config::validate` first; on failure, leave the running
   pool untouched.
2. Bump a generation counter and emit
   `ServerEvent::ConfigReloaded { generation }`.
3. Yield once so the broadcast send is observable on the calling
   task's scheduler before the function returns.

The full pool-rebuild path (peer-table swap, ring rebuild) is
deferred to the runtime-loop stage; the validation + event
contract that the embed surface promises today is the long-lived
contract under SemVer.

## Open follow-ups (handed to later stages)

1. Real cross-process forwarding for `inject_request` plans that
   resolve to remote peers (today: in-process registry only).
2. `PrometheusMetricsSink` once a Prometheus dependency is
   approved.
3. `tokio_rustls`-backed `MutualTlsTransport` example, swapping
   the in-memory duplex.
4. Full pool-rebuild semantics inside `ServerHandle::reload`.
5. `Server::builder()` does not yet accept a `Box<dyn
   TransportListener>`; the design doc sketches that shape but
   the listener wiring belongs to the runtime stage. The
   `Transport` re-export plus the existing `io::reactor` accept
   loop are the seam.
